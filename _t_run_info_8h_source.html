9///
10/// \class TRunInfo
11///
12/// This Class is designed to store and run dependent
13/// information. It is used to store run numbers, existence of
14/// detector systems, reconstruction windows, etc. The
15/// TRunInfo is written alongside both the fragment and
16/// analysis trees.
17///
18/// TRunInfo designed to be made as the FragmentTree
19/// is created. Right now, it simple remembers the run and
20/// subrunnumber and sets which systems are present in the odb.
21///
22/// Due to some root quarkiness, I have done something a bit strange.
23/// The info is written ok at the end of the fragment tree process.
24/// \code
25/// root [1] TRunInfo *info = (TRunInfo*)_file0->Get("TRunInfo")
26/// root [2] TRunInfo::ReadInfoFromFile(info);
27/// root [3] info->Print()
28/// TRunInfo Status:
29/// RunNumber: 29038
30/// SubRunNumber: 000
31/// TIGRESS: true
32/// SHARC: true
33/// GRIFFIN: false
34/// SCEPTAR: false
35/// =====================
36/// root [4]
37/// \endcode
38///
39/// \author P.C. Bender, <pcbend@gmail.com>
40/////////////////////////////////////////////////////////////////
